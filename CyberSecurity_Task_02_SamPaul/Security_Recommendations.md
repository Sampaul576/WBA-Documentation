# Security Recommendations

Based on the risk assessment and threat analysis above, the following 10 recommendations are proposed to strengthen the college's cybersecurity posture.

1. **Enable Multi-Factor Authentication (MFA)**
   Why: Even if a password is stolen via phishing, MFA prevents the attacker from logging in without the second verification factor.

2. **Regular System Updates and Patch Management**
   Why: Most major breaches (e.g., WannaCry) exploit known vulnerabilities that already had patches available — staying updated closes these gaps.

3. **Strong Password Policies**
   Why: Reduces the risk of brute-force and credential-guessing attacks against student/faculty accounts.

4. **Employee and Student Security Awareness Training**
   Why: Most breaches start with human error (clicking a phishing link); trained users are the first line of defense.

5. **Regular Data Backups**
   Why: Ensures that even if ransomware or hardware failure occurs, data can be restored without paying attackers or losing records permanently.

6. **Network Segmentation**
   Why: Limits how far malware or an attacker can move if one part of the network is compromised (e.g., isolating exam servers from general Wi-Fi).

7. **Role-Based Access Control (Least Privilege)**
   Why: Ensures users only have access to what they need, reducing the impact of insider threats or compromised accounts.

8. **Firewall and Intrusion Detection/Prevention Systems (IDS/IPS)**
   Why: Provides a monitored barrier between the college network and external threats, catching suspicious activity early.

9. **Regular Security Audits and Vulnerability Assessments**
   Why: Identifies weaknesses (like unpatched systems or misconfigurations) before attackers do.

10. **Incident Response Plan**
    Why: Ensures the college can react quickly and in an organized manner during a breach, minimizing downtime and damage instead of reacting in panic.
